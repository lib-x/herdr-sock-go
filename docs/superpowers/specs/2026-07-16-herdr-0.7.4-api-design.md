# Herdr 0.7.4 Go Socket API Adaptation Design

## Status

Approved for direct implementation. The user asked to adapt `herdr-sock-go`
after the WebShell 0.5.33 release, without another confirmation round.

## Goal

Update the public Go package from the Herdr 0.7.3 schema to the JSON portion
of Herdr 0.7.4 while preserving source and wire compatibility for existing
0.7.3 consumers.

## Selected approach

Use an additive compatibility release:

- add the five new JSON method constants and typed helpers;
- add metadata token and terminal-title fields to existing response types;
- add workspace metadata reporting and pane token patches;
- add the two new subscription helpers and event payload types;
- expose popup-capable plugin pane parameters with validated cell/percentage
  sizing; and
- retain removed `CustomStatus` fields with Go deprecation comments so
  existing programs still compile.

The raw `pane.graphics.stream` protocol is excluded. It is a dedicated binary
socket protocol and is not part of the newline-delimited JSON schema that this
package owns.

## Version and compatibility model

Herdr 0.7.3 and 0.7.4 both use protocol `16`, so `CurrentProtocol` remains 16.
The package will add `CurrentHerdrVersion = "0.7.4"` to identify the source
schema used to generate its types.

An application that only calls 0.7.3 methods remains compatible with a 0.7.3
server. Calling a new 0.7.4 method against a 0.7.3 server returns Herdr's
normal method error. Decoding 0.7.3 payloads remains safe because all new
response fields are optional maps or pointers.

Removed upstream fields are retained only for source compatibility:

- `PaneInfo.CustomStatus`
- `AgentInfo.CustomStatus`
- `PaneAgentStatusChangedEvent.CustomStatus`
- `PaneReportAgentParams.CustomStatus`
- `PaneReportMetadataParams.CustomStatus`
- `PaneReportMetadataParams.ClearCustomStatus`

Each receives a `Deprecated:` comment explaining that Herdr 0.7.4 no longer
produces or applies it. New code should use semantic `AgentStatus`, `Message`,
`StateLabels`, `Title`, `DisplayAgent`, and `Tokens`.

## Public JSON additions

### Methods

- `workspace.report_metadata`
- `pane.graphics.set`
- `pane.graphics.clear`
- `pane.graphics.info`
- `popup.close`

### Metadata

`WorkspaceInfo`, `PaneInfo`, and `AgentInfo` gain `Tokens map[string]string`.
`PaneInfo` and `AgentInfo` gain optional `TerminalTitle` and
`TerminalTitleStripped` fields.

`WorkspaceReportMetadataParams` and `PaneReportMetadataParams` use
`map[string]*string` token patches: a non-nil pointer sets a value, a nil
pointer serializes as JSON `null` and clears the key, and an omitted map leaves
all tokens unchanged.

### Events

Add helpers and payloads for:

- subscription `workspace.metadata_updated` with
  `WorkspaceMetadataUpdatedEvent`;
- subscription `pane.updated` with `PaneUpdatedEvent`.

The package does not automatically mix these subscriptions into older clients;
callers opt in explicitly, so 0.7.3 subscription requests remain unchanged.

### Graphics

Expose the JSON graphics API with:

- `PaneGraphicsFormat` constants for `png`, `rgb`, and `rgba`;
- `PaneGraphicsPlacementParams`;
- `PaneGraphicsSetParams` using `DataBase64`;
- `PaneGraphicsInfo` containing host cell dimensions; and
- typed `SetPaneGraphics`, `ClearPaneGraphics`, and `GetPaneGraphicsInfo`
  client methods.

The 512 KiB decoded-image limit and format-specific byte validation remain
server-owned. The Go client preserves its existing boundary behavior and
returns structured Herdr errors.

### Popup plugin panes

Expose `PluginPanePlacement` including the new `popup` value, typed plugin pane
open/focus/close params and results, and `ClosePopup`.

`PopupSize` has unexported representation and constructor functions:

- `PopupCells(uint16)` for terminal cell counts;
- `PopupPercent(uint8) (PopupSize, error)` for 1–100 percent.

Its JSON marshaler emits either an integer or a string such as `"80%"`. This
makes invalid percentage strings impossible to construct through the typed API.

## Error semantics

- Existing `Call` and `CallRaw` error behavior is unchanged.
- Invalid popup percentages fail locally from `PopupPercent`.
- New server method errors continue to decode as `*ErrorResponse`.
- No new validation is added to unrelated existing request structs.

## Documentation and release

README and package comments will identify Herdr 0.7.4/protocol 16, explain that
0.7.3 remains usable for the older method set, and include short metadata and
popup examples.

Release as annotated tag `v0.7.4`, matching the upstream schema target.

## Verification

- JSON marshal tests for token set/clear patches and popup sizes.
- Decode tests for tokens, terminal titles, workspace metadata events, and pane
  updated events.
- Socket fixture tests for workspace metadata, graphics info/set/clear, plugin
  popup open, and popup close.
- Existing tests remain unchanged except target-version fixtures.
- `gofmt`, `go vet ./...`, `go test ./...`, and `go test -race ./...` pass.

## Non-goals

- No raw `pane.graphics.stream` client.
- No Herdr config editor or `ui.copy_on_select` API; that setting is not a
  SockAPI method.
- No removal or renaming of existing public fields or methods.
- No attempt to emulate 0.7.4 methods on a 0.7.3 server.
